如何实现字体描边？

Outline effect to text

下面是一些比较没用的解决方案。

前置知识：描边的三种方式

在 ps 中，可以选择描边方式，分别有：外描边、居中描边以及内描边。

常见的实现字体描边方式

svg

svg 实现字体描边很简单，通过 <text> 搭配 fill 和 stroke 属性即可。

canvas

现在我们要祭出大杀器了：canvas。

在 canvas 中绘制文字的 api 有两个：

  • fillText(text, x, y [, maxWidth]): 在指定的(x,y)位置填充指定的文本，绘制的最大宽度是可选的
  • strokeText(text, x, y [, maxWidth]): 在指定(x,y)位置绘制带边框的文本，绘制的最大宽度是可选的。

首先要明确一点，在 canvas 中文字的描边方式也是居中描边！

对此，我们的解决办法是先画文字的边框，再在原位置上画文字。后面画的文字会把居中描边中向文字内部延伸的部分给盖住。因此，如果想让文字具有 1px 的边框，则画文字边框时宽度应为 2px。上代码：

总结

而无论是 css、svg 还是 canvas，文字描边的方式都是居中描边，且暂时没有属性来修改描边方式。

一般情况下，描边方式对于展示的影响不是很大，但效果在 stroke-width 变粗时就不是很好了，居中描边中内描边的部分会阻挡文字的正常显示。

上图中，两行字的字体大小一样，仅 stroke-width 发生了变化。

对于 css 和 svg 而言，这个问题暂时是无解的。但对于 canvas 而言，这个问题可以通过我们上面提到的方法来完美解决。

附录

在调研过程中还遇到了一些别的解决字体描边内描边过大的方案，一并放在这里了：

  • 调大字号：调大字号后，中间的部分也会变宽。

  • 如果只显示数字或字母的话，可以让视觉同学将每个数字或字母都单独切成图，显示的时候根据内容做拼接。

  • 还有一个属性叫 paint-order，可以指定是先绘制 fill 还是先绘制 stroke，不够支持率十分感人，根本不能用。
